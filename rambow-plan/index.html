02The chase

Three jobs. Nothing else.

Rambow does exactly three things, in priority order. Everything in this plan traces to one of them.

Know what you own

Every card from the spreadsheet, browsable and searchable, each with a photo. Serial numbers, print runs, and grades preserved exactly.

See every rainbow

For each card-set (player + year + product + card #), a completion meter: parallels owned vs. parallels that exist, with the missing ones named.

Close the gaps

Live eBay search on any missing parallel, plus a watchlist that pings your phone when a listing appears under your target price.

Featured chases AD 99 · MF 28

Aaron Donald and Marshall Faulk each get a hero page pinned to the top of the app. All nine players get the same rainbow mechanics — featured just means these two lead the dashboard and get first claim on checklist research and imagery.

03The chase

The rainbow engine

A rainbow is only measurable against a master checklist — the full list of parallels that exist for a card. Your inventory says what you have; the checklist says what "done" means. Rambow seeds checklists from published sources: Cardboard Connection, Beckett, and Panini's official checklists, verified per product and stored per card-set with print runs. eBay listing discovery fills gaps over time (a parallel showing up in listings that isn't on the checklist gets flagged for review).

Completion math

Completion = distinct parallels owned ÷ parallels on the checklist. Each parallel is tiered — base numbered short print /25 and under 1/1 — so a rainbow can be "complete except 1/1s," which is its own badge (chasing a 1/1 someone else owns is a different sport).

The 50% rule Your call, encoded

Any rainbow past 50% completion is promoted to an active chase: its missing parallels get reference images pulled from eBay, appear on the dashboard's "closest to done" rail, and are eligible for watchlist alerts. Below 50%, a rainbow is just a tally — no image fetching, no noise.

04Find & buy

The hunt. You pull the trigger.

Both buy-side modes you picked, powered by the eBay Browse API — the same integration already running in your morning report (JBCardSearch keyset), so credentials and query patterns are proven. Hard guardrail: Rambow never buys anything. Every path ends in a deep link to the eBay listing where you tap Buy yourself.

Mode 1 · Watchlist + price alerts

Set target price on a missing parallel → Nightly scan queries eBay per watch → Match found under target → ntfy push to your phone → Tap → eBay listing → you buy

Same ntfy channel pattern as the morning report. Each watch stores the exact query (player, year, product, card #, parallel, serial hints like "/25"), a target price, and a raw/graded preference. Alerts include title, price, condition, thumbnail, and the listing link.

Mode 2 · In-app live search

Tap any missing parallel → Live eBay query, pre-built & tuned → Listings ranked: exact match first, price ascending → Open listing on eBay

Query quality is the whole game. Card-title matching is fuzzy ("Tie-Dye /25" vs "Tie Dye 25/25" vs "SSP tie dye"). Each parallel gets a curated query template plus negative keywords (lot, reprint, custom, digital), refined as bad matches appear. A "not a match" button on any result feeds the filter.

05Find & buy

Every card gets a face

A rainbow page with 26 gray placeholders isn't a rainbow. Two image pipelines fix that — one for what you own, one for what you're missing.

Owned cards — your photos

A phone-friendly capture flow: open Rambow on your phone, tap a card, shoot front (back optional) — stored in Supabase Storage, auto-cropped and linked. A "needs photo" queue works through the backlog worst-first (featured chases → graded slabs → everything else). For slabbed cards, the cert number on the label doubles as verification. Until you shoot a card, an eBay stock image of the same parallel stands in, clearly badged as a stand-in.

Missing cards — reference images

For every active chase (the >50% rainbows, per the 50% rule), missing parallels get reference images pulled automatically from eBay listing photos via the Browse API. So the rainbow grid shows the actual Tie-Dye you're hunting, not a question mark. Below 50%, no fetching — keeps API calls and storage focused on chases that are real.

09The product

Every term, plain English

This glossary ships inside the app as tooltips on every term below.

RainbowEvery color parallel of one card (same player, year, product, card #). Completing one is the point of this app.
ParallelA variant of a base card printed in a different color/pattern (Tie-Dye, Zebra, Gold), usually rarer than base.
Prizm / Select / MosaicPanini's chromium card brands — the products where parallels (and rainbows) live.
Serial numbered (/25)Card stamped with its copy number out of a fixed print run, e.g. 11/25. Lower run = rarer.
Print runTotal copies made of a parallel. The "Qty Manufactured" column in the spreadsheet.
1/1A one-of-one — the only copy in existence. The grail tier of any rainbow. You own 14.
SP / SSPShort print / super short print — parallels with no stamped number but known tiny supply (White Sparkle, Kaboom).
RawAn ungraded card, no case. Cheaper, but condition is your own judgment.
SlabA card professionally graded and sealed in a labeled case (PSA, BGS, SGC), with a cert number.
PSA 10 / BGS 9.5Grades — a 1-to-10 condition score from a grading company. Gem Mint 10 commands the premium.
CompA recent sold price for the same card — what the market actually pays, vs. what sellers ask.
eBay Browse APIeBay's official search API — how Rambow finds live listings without scraping.
Master checklistThe verified list of every parallel that exists for a card-set. What your rainbow is measured against.
Active chaseA rainbow past 50% completion — gets missing-card images, dashboard placement, and alert eligibility.

10Build

Stack & seed

Same stack shapes you already run — nothing exotic, nothing to babysit.

Frontend

React + Vite, mobile-first (the photo flow lives on your phone). Rams theme: royal navy chrome, Sol accents, Bone canvas, Archivo display type.

Data

Supabase — Postgres for cards, checklists, watches, alerts; Storage for card photos and reference images. Single user, simple auth.

Jobs & hosting

DigitalOcean App Platform, auto-deploy from GitHub. Nightly scan job hits eBay Browse (existing JBCardSearch keys) and pushes via ntfy.

Data model Five tables

TableWhat it holds
card_setsPlayer + year + product + card # — the unit a rainbow is measured on (207 seeded)
parallelsMaster checklist rows per card_set: name, tier, print run, source URL, status (verified / estimated / ebay-flagged)
cardsYour inventory (593 seeded): parallel link, serial #, grade, purchase price, notes, photo refs
watchesMissing-parallel watches: query template, negative keywords, target price, raw/graded pref, status
alertsEvery match ever pushed: listing id, price, thumbnail, outcome (bought / passed / junk)

The import script is idempotent: re-running against an updated spreadsheet diffs before writing (UC-02). The spreadsheet stays your editing surface as long as you want it to be — the app can take over as source of truth whenever adding cards in-app feels easier.
